# mod_micp — AI-Powered Interactive Lessons for Moodle

[**中文版**](./README_zh.md)

> **The old way:** Teachers spend hours building interactive exercises.
> **The mod_micp way:** Tell an AI the topic. Get a complete interactive lesson. Done.

mod_micp has one job: make interactive HTML lessons that grade themselves in Moodle — generated by AI, served by the plugin.

---

## The Core Loop

```
You → "Create a lesson about [topic]" → AI generates a ZIP
                                                    ↓
                                      Upload to mod_micp
                                                    ↓
                              Students interact → server grades → score in gradebook
```

**The AI does the authoring.** The plugin handles delivery and grading.

---

## Try It Now

### 1. Install the plugin

```bash
cp -r mod/micp /path/to/your/moodle/mod/
```
Visit **Site Administration → Notifications**. Requires **Moodle 5.x**, PHP 8.1+.

### 2. Use a ready-made sample (zero authoring)

Three pre-built lessons are included — upload and go:

| File | Topic |
|---|---|
| `generated/photosynthesis-micp.zip` | Photosynthesis — progressive steps |
| `generated/audio-digitization-micp.zip` | Audio digitization — 33 interactions (EN) |
| `generated/audio-digitization-micp-zh.zip` | Same, Chinese |

### 3. Generate your own with the bundled Skill

This repo ships with **`micp-html-authoring`** — an AI agent Skill that knows how to build mod_micp lesson packages.

**OpenCode agent trigger:**
```
"Create a MICP lesson about [any topic you teach]"
```

The AI will generate:
- `index.html` — the interactive experience
- `micp-scoring.json` — server-side grading rules
- `assets/` — bundled JS runtime

ZIP it → upload to mod_micp → students get a self-grading lesson.

Full Skill documentation: [`.skills/micp-html-authoring/SKILL.md`](.skills/micp-html-authoring/SKILL.md)

## How It Works

```
AI generates package          Plugin delivers lesson
    ↓                              ↓
[index.html] + [micp-scoring.json]  →  Student opens activity
                                         ↓
                              Student interacts
                              MICP.sendEvent() → server
                                         ↓
                              Student clicks "Done"
                              MICP.submit() → scoring
                                         ↓
                              grade_update() → Moodle gradebook
```

- **Client SDK** (`window.MICP`): fires events, submits — never computes a score
- **Server-side scoring**: reads `micp-scoring.json` rules, returns grade
- **Gradebook**: via Moodle's official `grade_update()` API

---

## Why This Exists

Teachers shouldn't be template builders. They should be curriculum designers.

mod_micp separates concerns:
- **You** decide what students should experience and learn
- **AI** builds the interactive pages
- **The plugin** delivers and grades — reliably, at scale

### Lesson Package Structure

```
my-lesson.zip
├── index.html          # Interactive HTML (generated by AI)
├── micp-scoring.json   # Grading rules
└── assets/
    └── micp.js         # Required runtime
```

### `micp-scoring.json` (generated by AI)

```json
{
  "rules": [
    {
      "id": "q1_choice",
      "label": "Question 1",
      "check": { "event": "interaction", "scoring": { "correct": "a" } }
    }
  ],
  "scoring": { "strategy": "all_or_nothing" }
}
```

Without this file: **100 if any interaction recorded, 0 otherwise**.

### Client SDK

```javascript
MICP.init();                                          // auto on load
MICP.sendEvent('interaction', { interactionid: '...', response: '...', outcome: '...' });
MICP.submit({ raw: { actions: actions } });           // on submit click
const ctx = MICP.getContext();                         // { cmid, userid, sesskey, ... }
```

### Scoring Strategies

- `all_or_nothing` — all rules satisfied = 100, else 0
- `proportional` — partial credit by rule weight

### Capabilities

| Capability | Default role | Description |
|---|---|---|
| `mod/micp:addinstance` | teacher | Create/edit activities |
| `mod/micp:view` | student | View and interact |
| `mod/micp:submit` | student | Submit and receive grade |
| `mod/micp:viewreports` | teacher | View grade report |

### Architecture

```
mod/micp/
├── lib.php                 # Scoring engine, gradebook wrapper
├── view.php                # Activity page (iframe host)
├── file.php                # Pluginfile handler for lesson assets
├── report.php              # Participant grade report
├── micp.js                 # Client SDK
├── db/install.xml          # Tables: micp_events, micp_submissions
├── db/services.php         # Moodle AJAX endpoints
├── classes/local/
│   └── scoring_service.php  # Server-side grading logic
└── lang/en/micp.php
```
