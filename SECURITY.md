# Security Policy

## Reporting a Vulnerability

Do not report security vulnerabilities in public issues.

Use one of the following private channels instead:

- the repository's private security advisory workflow, if enabled
- the maintainer contact published with the plugin release or Moodle Plugins Directory entry

When reporting a vulnerability, include:

- affected version
- Moodle version
- PHP version
- clear reproduction steps
- expected impact

## Disclosure

Please allow time for the issue to be reproduced, fixed, and prepared for release before public disclosure.
